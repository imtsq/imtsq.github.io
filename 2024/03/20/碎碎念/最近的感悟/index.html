cdn

大致知道了cdn运行流程，在找了很多资料发现，免费的还是有限，

7牛云：虽然能用做图床，或者静态托管，但是要域名，难白嫖

其他国外的静态托管，还不如自己GitHub + 飞机，

最后自己选定了博客静态管理组合

  • GitHub（静态托管
  • GitHub + PicGo 作为图床
  • jsdelivr 对图床cdn加速

或许后面会结合gitee是托管网站，不如容易抽风，图床容易失败，静态托管听说一般，博客园额，加入vip的哪个时刻，我觉得还是应该有个自己的博客

有点像老美当时对中国GPS的制裁，然后查阅了很多的免费博客，发现都有恰米元素SVIP vip，还是自己创建比较方便，当然也能用GitHub直接作为笔记md文档，

和文件夹差不多格式

对域名的思考

有个域名确实访问，简单，但是也简介不到多少，很少字符贵，不如GitHub自己的，

最近学习

在做了大量项目的感悟，确实感觉java没啥感觉了，感觉属于个人创作的比较少，可能最有设计的就是数据库的设计，其实感觉

大部分工作都在写前端，感觉前端写起确实又创造性，在搭建博客我个人感觉有时候还是应该学点脚本语言，不是为了装逼

更多是为了简化低代码的工作，纯浪费时间，比如hexo new articleName 然后我就要 hexo clear….. hexo d

其实写个简单的脚本界面，搞个python直接图形化，然后做个和picGo差不多上传图片，一键上传感觉能简化很多工作，要是

项目做大了真正做个md编辑工具开源，完全可以上线了，

对于hexo博客的思考

个人感觉还是又很大问题，但是人无完人，当我链接先关地址或者做个目录的适合，链接本地文件但是可能在文章有点不兼容

大概就是我指向另一篇文章链接在网页端就失效了
